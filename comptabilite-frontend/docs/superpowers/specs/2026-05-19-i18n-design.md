# i18n Multi-Language Design (EN / FR)

**Date:** 2026-05-19
**Status:** Approved

## Overview

Add runtime English/French switching to the Angular 17 facturation app using `@ngx-translate/core`. Language is detected from the browser on first visit, persisted to `localStorage`, and toggled via a button in the top header.

## Architecture & Setup

- **Library:** `@ngx-translate/core` + `@ngx-translate/http-loader`
- **Registration:** `TranslateModule` configured in `app.config.ts` via `HttpLoaderFactory` pointing to `src/assets/i18n/`
- **Translation files:** `src/assets/i18n/en.json` and `src/assets/i18n/fr.json`
- **`LanguageService`** (`src/app/core/i18n/language.service.ts`):
  - Resolves initial language: `localStorage["lang"]` → `navigator.language` (mapped to `"en"` or `"fr"`) → `"fr"` fallback
  - Exposes `currentLang` signal
  - Exposes `setLanguage(lang: 'en' | 'fr')` — calls `TranslateService.use()` and persists to `localStorage`

## Language Switcher UI

- Located in `header.component.html`, visible on every page
- Single toggle button: shows flag + code of the **current** language (🇬🇧 EN or 🇫🇷 FR)
- One click switches to the other language — no dropdown needed for 2 languages
- `HeaderComponent` injects `LanguageService`, calls `setLanguage()` on click, reads `currentLang` signal for button label

## Components to Update

All components that contain hardcoded strings need `TranslateModule` added to `imports[]` and strings replaced with `{{ 'KEY' | translate }}`:

| Component | File |
|-----------|------|
| HeaderComponent | `core/layout/header.component` |
| MainLayoutComponent | `core/layout/main-layout.component` |
| HomeComponent | `features/home/home.component` |
| LoginComponent | `features/login/login.component` |
| DashboardVenteComponent | `features/dashboard-vente/dashboard-vente.component` |
| InvoicesComponent | `features/invoices/invoices.component` |
| ClientsComponent | `features/clients/clients.component` |
| NewClientComponent | `features/clients/new-client/new-client.component` |
| NewInvoiceComponent | `features/invoices/new-invoice/new-invoice.component` |

For dynamic strings in `.ts` files (e.g. status labels), use `TranslateService.instant()` or move the label resolution to the template.

## Out of Scope

- RTL support
- More than 2 languages
- Route-based locale (`/en/`, `/fr/`)
- Date/currency locale formatting (separate concern)
